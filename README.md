# Chrome DevTools — Crash Course
 
> Learning notes for using **Chrome DevTools** to inspect, debug, and optimize web pages — straight from the browser.
 
Chrome DevTools is a set of web developer tools built into Google Chrome. This repo is my structured write‑up of how to use each panel productively: live‑editing pages, debugging JavaScript with breakpoints, inspecting network traffic, managing storage, and profiling performance.
 
📺 Based on the [freeCodeCamp.org Chrome DevTools Crash Course](https://youtu.be/gTVpBbFWry8) by **Nabheet Madan** · 📚 Official reference: [developer.chrome.com/docs/devtools](https://developer.chrome.com/docs/devtools)

## Open DevTools in 1 second
 
| Action | Windows / Linux | macOS |
|---|---|---|
| Open DevTools | `F12` or `Ctrl+Shift+I` | `Cmd+Option+I` |
| Inspect an element | `Ctrl+Shift+C` | `Cmd+Shift+C` |
| Jump to Console | `Ctrl+Shift+J` | `Cmd+Option+J` |
| Command Menu (run anything) | `Ctrl+Shift+P` | `Cmd+Shift+P` |
 
> The **Command Menu** (`Ctrl/Cmd+Shift+P`) is the single most useful shortcut — type what you want ("screenshot", "coverage", "disable cache") and run it.
 
## What's covered
 
| Panel | You use it to… |
|---|---|
| **Elements** | View/edit the live DOM and CSS, inspect the box model |
| **Console** | Run JavaScript, read logs/errors, query the page |
| **Sources** | Debug JavaScript with breakpoints, edit & persist files |
| **Network** | Inspect every request, headers, timing, throttling |
| **Application** | Local/session storage, cookies, IndexedDB, service workers |
| **Performance** | Record and analyze runtime performance |
| **Memory** | Find leaks with heap snapshots |
| **Security / Lighthouse** | Check HTTPS/certs and run quality audits |
